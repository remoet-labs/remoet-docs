# Get Started

## Sign up

Create a developer profile at [www.remoet.dev](https://www.remoet.dev). You can sign in with GitHub or Google.

## Generate an API key

Once signed in, head to [www.remoet.dev/api-keys](https://www.remoet.dev/api-keys). You can hold up to **5 keys per account** at any time. Each key works against both the MCP server and the REST API.

Daily request limits apply per account (not per key):

| Tier | REST API requests / day | MCP requests / day |
|---|---|---|
| Free | 300 | 30 |
| Pro | 5,000 | 150 |
| Max | unlimited | unlimited |

Limits reset daily UTC. See <Link href="/tiers">Tiers</Link> for the full plan breakdown.

## Use your key

Two ways to plug in:

### A. MCP server (for AI agents)

If you're connecting an AI agent — Claude, Cursor, Codex, Windsurf — see <Link href="/mcp">MCP</Link> for the install URL and per-client setup. This is the recommended path for agentic workflows.

### B. REST API (for your own code)

If you're building a portfolio site or pulling data into your own tooling, the REST API is what you want. Set the `Authorization: Bearer <Your key>` header on any of the endpoints below.

#### Endpoints

- <Link href="/hello">`GET /user/hello`</Link> — health check
- <Link href="/full">`GET /user/full`</Link> — everything in one shot
- <Link href="/links">`GET /user/links`</Link> — social links
- <Link href="/jobs">`GET /user/jobs`</Link> — work history
- <Link href="/projects">`GET /user/projects`</Link> — project cards
- <Link href="/education">`GET /user/education`</Link> — education entries
- <Link href="/linktrees">`GET /user/linktrees`</Link> — your link trees

All endpoints are served from `https://api.remoet.dev`. See <Link href="/get-started/authorization">Authorization</Link> for the header shape.
