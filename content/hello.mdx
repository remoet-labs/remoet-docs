# `GET/user/hello`

The `/user/hello` endpoint is a testing endpoint for veryfying that your authentication is setup correctly.
One might argue that any of the authenticated endpoints would suffice, but alas, this one exists as well.

## Basic
- type: `REST`
- method: `GET`
- url: https://www.api.remoet.dev/user/hello
- returns: `string`

## cURL
```cURL
curl --location 'https://www.api.remoet.dev/user/hello' --header 'Authorization: Bearer <Your key>'
```

## HTTP
```http
GET /user/hello HTTP/1.1
Host: www.api.remoet.dev
Authorization: Bearer <Your key>
```

## JavaScript Fetch
```javascript
const myHeaders = new Headers();
myHeaders.append("Authorization", `Bearer ${key}`);

const requestOptions = {
  method: "GET",
  headers: myHeaders
};

fetch("https://www.api.remoet.dev/user/hello", requestOptions)
  .then((response) => response.text())
  .then((result) => console.log(result))
  .catch((error) => console.error(error));
```

## Result
```
Hello from the public API! <Your email>
```

## Non authenticated response
```json
{
    "statusCode": 401,
    "message": "Invalid API Key",
    "error": "Unauthorized"
}
```
